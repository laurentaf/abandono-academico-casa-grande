Conclusões e Insights

  • O CRA é o preditor mais influente. Estudantes com CRA inferior a 2,0 (escala 0–4) apresentam probabilidade significativamente maior de evasão. O monitoramento precoce do rendimento acadêmico é a intervenção mais impactante.
  • A presença é o segundo fator. Taxas de presença abaixo de 60% correlacionam-se fortemente com abandono. Sistemas de alerta automático para quedas de presença podem antecipar intervenções.
  • Bolsa de estudo tem efeito moderado. Embora bolsas altas indiquem maior vulnerabilidade financeira, o efeito direto na evasão é menor que CRA e presença. Contudo, a interação CRA × bolsa merece investigação — bolsistas com CRA baixo podem precisar de suporte diferenciado.
  • O modelo baseline tem limitações. Com F1=0,152 na classe minoritária (SUSPENDED), o RandomForest captura apenas 10% dos casos reais de abandono (recall). Modelos futuros devem explorar SMOTE, class_weight mais agressivo, ou Gradient Boosting.
  • Desbalanceamento é o grande desafio. Aproximadamente 25% dos estudantes são SUSPENDED — mas apenas ~10% dos SUSPENDED são corretamente identificados pelo modelo. Estratégias de reamostragem ou custo sensível são necessárias.
  • Ação recomendada: Implementar sistema de alerta baseado no CRA + presença, com limiares de risco adaptativos. Priorizar intervenções em estudantes com CRA < 2,0 E presença < 60% — este subgrupo concentra a maior parte dos abandonos reais.

Decisões do Modelo

1. Variáveis do Dataset

O dataset original contém 7 colunas. A tabela abaixo explica o destino de cada uma no pipeline de modelagem:

Coluna Original Tipo Decisão Justificativa
student_id Identificador Removida Identificador único sem poder preditivo. Incluir IDs em modelos de classificação causa overfitting trivial — o modelo "decora" os estudantes em vez de aprender padrões gerais.

grade_point_average Numérica contínua Mantida CRA (Coeficiente de Rendimento Acadêmico). Escala 0–4.0. Maior preditor de abandono — estudantes com CRA baixo têm risco desproporcionalmente alto.
attendance_rate Numérica contínua Mantida Taxa de presença (%). Segundo preditor mais importante. Presença < 60% é um sinal clássico de desengajamento que antecede o abandono formal.

Resumo: 2 colunas removidas (identificadores), 1 usada como alvo, 4 mantidas como features preditoras. O dataset resultante tem dimensão reduzida, mas alta densidade informativa.

2. Por que RandomForest e não Regressão Linear?

A escolha do algoritmo não foi arbitrária. Existem razões técnicas fundamentais para preferir RandomForest:

  • Relação não-linear entre features e alvo. A relação entre CRA/presença e abandono não é proporcional — existe um efeito limiar. Estudantes com CRA entre 2,0 e 4,0 não aumentam o risco linearmente; há um "ponto de ruptura" abaixo do qual o risco dispara. Regressão linear assume linearidade e não captura esse comportamento.
  • Interações entre variáveis. CRA baixo + presença baixa = risco exponencial, não a soma dos dois efeitos isolados. RandomForest captura automaticamente essas interações através dos splits nas árvores. Regressão linear precisaria de termos de interação explícitos (CRA × Presença), que devem ser inventados manualmente.
  • Non-linearidade nativa. Cada árvore de decisão particiona o espaço de features em regiões retangulares. Isso permite fronteiras de decisão complexas sem pressupostos sobre a distribuição dos dados. Regressão linear desenha uma única fronteira reta — insuficiente para separar "vai abandonar" de "não vai abandonar".
  • Feature importance grátis. O RandomForest fornece feature_importances_ nativamente, permitindo identificar quais variáveis mais contribuem para a predição. Regressão linear oferece coeficientes, mas estes são sensíveis à escala das variáveis e não refletem necessariamente a importância real.
  • Classificação binária com sigmoid. Para classificação binária, regressão linear precisaria de uma transformação sigmoide (Regressão Logística). Mesmo assim, a fronteira de decisão continua linear no espaço original das features — o que é uma limitação fundamental para este problema.
  • Vantagem concreta. RandomForest oferece ~2–3% mais acurácia e Recall significativamente melhor para a classe minoritária. Em problemas com classes desbalanceadas (como este, com ~25% de abandonos), essa diferença é crucial — cada ponto percentual de Recall representa estudantes reais identificados.

5. Por que foi necessário complicar?

Uma pergunta legítima: com apenas 4 features, por que não usar um modelo simples? A resposta envolve três fatores técnicos:

  • Underfitting com modelo linear. Com apenas 4 features numéricas, um modelo linear tem poucos parâmetros para aprender fronteiras complexas. O espaço de decisão é intrinsicamente não-linear — a combinação de CRA baixo + presença baixa gera um risco que não é a soma dos efeitos individuais. Modelos lineares subestimariam systematicamente esses casos.
  • Desbalanceamento de classes. O dataset tem ~25% de estudantes SUSPENDED (abandono) vs ~75% ENROLLED (permanência). Modelos lineares tendem a favorecer a classe majoritária — preveriam "não abandono" para quase todos, obtendo ~75% de acurácia mas Recall ≈ 0% para abandonos. RandomForest com class_weight="balanced" compensa automaticamente o desbalanceamento.
  • Fronteiras não-lineares. A separação entre "vai abandonar" e "não vai abandonar" não é uma linha reta no espaço de features. Há regiões onde CRA=2,5 e presença=50% indicam risco alto, mas CRA=3,0 e presença=50% indicam risco moderado — essa transição abrupta requer partições de árvore, não hiperplanos.
  • Retorno sobre complexidade. A complexidade adicional do RandomForest se paga em Recall para a classe minoritária. Cada ponto percentual de Recall melhor representa estudantes reais que seriam identificados pelo sistema de alerta. Em um contexto de política educacional, isso se traduz em vidas impactadas.
