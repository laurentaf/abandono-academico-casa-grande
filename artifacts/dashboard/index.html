Conclusões e Ações

  • Queda de Atividade (Semana 3): Estudantes com last_activity_day ≤ 21 + clicks < 500 — alerta automático no VLE + contato do tutor. Ação de baixo custo que pode reverter o desengajamento precoce.
  • Mentoria para Baixa Submissão: Estudantes com assessment_count ≤ 2 até o meio do módulo — mentor dedicado 1h/semana. A feature assessment_count é a 2ª mais importante (~12%).
  • Nudge de Consistência: Estudantes com submission_rate < 0,30 (proporção de entregas) — e-mail automatizado + lembrete no VLE a cada nova avaliação. Custo baixo, efeito gradual.

⚠ Avisos e Limitações

  • Correlação não é causalidade. O modelo é PREDITIVO, não CAUSAL. Baixa atividade VLE é um sinal de desengajamento, não a causa da evasão. Intervenções devem focar em reverter o desengajamento, não em "corrigir" a feature.
  • OULAD = 1 universidade UK (2013–2014). Dados de uma única instituição de ensino a distância no Reino Unido (pré-pandemia). Não generalizável diretamente para ensino presencial, outros países, ou contextos pós-2020. Recalibração necessária para outras instituições.
  • Proxy linear nos 3 indicadores (Logistic Regression com last_activity_day, assessment_count, submission_rate) perde ~0,02 ROC-AUC vs Random Forest completo. A simplificação é intencional para permitir cálculos analíticos de esforço por indicador em file:// sem servidor.
